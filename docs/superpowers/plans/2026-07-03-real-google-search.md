# План: включить реальный поиск Google в Croogle

**Статус кода:** реализация уже есть. `app/api/search/route.ts` вызывает Google
Custom Search JSON API, нормализует ответ в `{title, link, snippet}[]` и при
ошибке/отсутствии ключей отдаёт демо-кэш (`lib/fallback.ts`). Чтобы пошла живая
выдача — нужно только выдать роуту валидные ключи. **Правок кода не требуется.**

Как это работает сейчас:
- Есть `GOOGLE_API_KEY` + `GOOGLE_CX` → роут зовёт Google, `"fallback": false`, реальные результаты.
- Нет ключей / ошибка / лимит → `"fallback": true`, демо-кэш (демо не падает).

---

## Шаг 1 — Создать Programmable Search Engine (даёт `cx`)  · делает человек

1. Открыть https://programmablesearchengine.google.com/controlpanel/create
2. Имя — любое (напр. «Croogle»). В «What to search» выбрать **Search the entire web**.
3. Create → на странице движка скопировать **Search engine ID** — это `GOOGLE_CX`.

## Шаг 2 — Получить API-ключ  · делает человек (нужен Google-аккаунт)

1. Открыть https://developers.google.com/custom-search/v1/introduction
2. Нажать **Get a Key**, выбрать/создать проект → скопировать **API key** — это `GOOGLE_API_KEY`.
3. (Опц., для безопасности) в Google Cloud Console ограничить ключ по API «Custom Search API».

## Шаг 3 — Прописать ключи локально  · могу сделать я, как только пришлёшь значения

Вписать в `.env.local` (файл гитигнорится — в репозиторий не попадёт):
```
GOOGLE_API_KEY=<ключ из шага 2>
GOOGLE_CX=<id из шага 1>
```

## Шаг 4 — Проверить живую выдачу  · сделаю я

```bash
npm run dev
curl "http://localhost:3000/api/search?q=nfactorial"
```
Ожидаемо: JSON с реальными результатами Google и `"fallback": false`.
Плюс визуальная проверка на `/search` (скриншот).

## Шаг 5 — Проверить быстрый тест на срыв лимита  · сделаю я

Бесплатный лимит Custom Search — **100 запросов/день**. Для демо этого хватает,
а `fallback` страхует, если лимит кончится прямо на сцене. Ничего настраивать не
нужно — просто помним про заготовленный демо-запрос.

---

## Границы / что НЕ делаем
- Не убираем `fallback` — это демо-страховка (правило хакатона: демо не должно падать).
- Ключ в серверном роуте не светится во фронте (env только на сервере). Для будущего
  статического хостинга (GitHub Pages) пришлось бы выносить вызов в браузер и класть
  ключ в `NEXT_PUBLIC_*` — но мы на серверной архитектуре, так что это не нужно.
